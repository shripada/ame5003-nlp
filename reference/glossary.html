NLP Glossary

The canonical vocabulary for both courses. Once a term is defined here, every lesson uses it this way — consistency is worth more than elegance.

How to use this

Print it. Hand it out in week 1. Students who lack the nomenclature cannot ask good questions, and NLP is unusually dense in near-synonyms that mean subtly different things (type vs token, lemma vs stem). Terms marked like this are ones students reliably confuse — and which examiners reliably probe.

The basics — used from week 1, taught properly from week 7

Model
A program that learned its behaviour from examples rather than from rules someone wrote. Opposite of a regex, where you type the rule out yourself. Keeps a set of numbers for each word it knows, and adjusts them while learning. Built properly from session 20. Gentle intro.
Training
Showing a model examples so it can work the patterns out. Done once, in advance. Contrast inference: using the finished model on something new.
Parameters
The numbers inside a model that training adjusts. "A bigger model" means more of these.
Rules vs. learning
The spine of this whole course. Rules: a person writes down what to do — regex (weeks 1–2), VADER's word list (lab 1). Precise, fast, and someone must think of every case. Learning: show examples, the machine works it out — Naïve Bayes (week 6) onward. Handles cases nobody listed, needs data and gives no guarantees. Nearly every "why do we still teach the old way?" question is answered by this row.
Vocabulary
The fixed list of every token a model knows. Decided before training and frozen. One row of parameters per entry, so vocabulary size drives model size. Anything not in it becomes <UNK> — which is the problem subword tokenization exists to solve.

Unit I — Text processing & IR

Corpus pl. corpora
A body of text used for training or evaluation.
Type vs. Token
A token is an instance; a type is a distinct entry in the vocabulary. "the cat sat on the mat" = 6 tokens, 5 types ("the" repeats). Vocabulary size means types. exam favourite
Tokenization
Splitting raw text into tokens. Not a string operation — a modelling decision. See Lesson 03.
Normalization
Collapsing surface variants to one form: case-folding, Unicode normalization, U.S.A. → USA. Deliberately destroys information you judge irrelevant.
Stemming vs. Lemmatization
Stemming chops affixes by rule — fast, crude, output need not be a real word (studies → studi). Lemmatization maps to the dictionary headword using morphology + POS — slower, correct (studies → study, better → good). the single most-confused pair in Unit I
Lemma
The dictionary/citation form. run is the lemma of ran, running, runs.
Stop words
High-frequency, low-content words (the, is, of) often removed. Note: harmful for tasks where they matter — "to be or not to be" is entirely stop words.
Subword tokenization
Units smaller than words, so unseen words decompose instead of becoming <UNK>. See BPE.
BPE Byte-Pair Encoding
Learns subword units by repeatedly merging the most frequent adjacent symbol pair. Merge count k sets vocabulary size. Origin: a 1994 compression algorithm.
WordPiece
BPE variant used by BERT: merges the pair that most increases training-data likelihood, not the most frequent one. The ## prefix marks "continues previous word".
<UNK>
The out-of-vocabulary catch-all token. Its existence is the problem subword tokenization solves.
POS tagging Part-of-Speech
Labelling each token with its grammatical category (noun, verb…). A sequence-labelling task.
NER Named Entity Recognition
Finding and typing proper-name spans (PERSON, ORG, LOC). Span-based, so harder than POS: entities cover multiple tokens.
Term-document incidence matrix
Binary matrix, terms × documents: 1 if term occurs in doc. Conceptually clean, hopelessly sparse at scale — which motivates the inverted index.
Inverted index
Maps each term → postings list of documents containing it. "Inverted" because the natural direction is doc → terms. The data structure behind every search engine.
Postings list
The list of doc IDs for one term, kept sorted so queries can be answered by merging lists.

Unit II — LM, sentiment, embeddings

Language model LM
Assigns probability to a token sequence — equivalently, predicts the next token.
N-gram
A contiguous run of n tokens. An n-gram LM applies the Markov assumption: only the previous n−1 tokens matter.
MLE Maximum Likelihood Estimate
For n-grams: P(w|context) = count(context,w) / count(context). Just counting and dividing — and it assigns probability zero to anything unseen, which is why smoothing exists.
Smoothing
Moving probability mass to unseen n-grams so nothing is impossible. Laplace/add-one is the teaching example; Kneser-Ney is the good one.
Perplexity PP
Standard intrinsic LM metric: the inverse probability of the test set, normalised by token count — equivalently 2cross-entropy. Intuition: the model's average branching factor. Lower is better. Students routinely get that backwards. Only comparable across models with identical vocabularies.
Naïve Bayes
Generative classifier applying Bayes' rule with a conditional-independence assumption across features. SLP3 Appendix B — not ch 4
The "naïve" assumption
That features (words) are conditionally independent given the class. It is false for language — and the classifier works well anyway. Being able to say why we tolerate a known-false assumption is the whole point of the topic.
BoW Bag of Words
Represent a document by word counts, discarding order. "Dog bites man" = "man bites dog".
TF-IDF
Weight = term frequency × inverse document frequency. TF rewards words frequent in this doc; IDF (a log ratio) penalises words common across all docs. Together: words that are distinctive here. The log in IDF is the exam question — it damps the effect so rarity helps sub-linearly.
Cosine similarity
Similarity as the cosine of the angle between vectors: dot product over the product of magnitudes. Ignores length, so a long and a short doc on the same topic still match.
Sparse vs. Dense vectors
Sparse (BoW, TF-IDF): dimension = vocabulary size, mostly zeros, each dimension is an interpretable word. Dense (Word2Vec, GloVe): a few hundred dimensions, all non-zero, individually uninterpretable. The move from sparse to dense is the central arc of Unit II.
Distributional hypothesis
"You shall know a word by the company it keeps" (Firth, 1957). Words in similar contexts have similar meanings. The premise every embedding method rests on.
Embedding
A learned dense vector for a word/token. A lookup table — vocabulary × dimensions — trained by gradient descent, not hand-built.
CBOW vs. Skip-gram
The two Word2Vec architectures, and they are mirror images. CBOW: context → predict the centre word. Skip-gram: centre word → predict the context. Skip-gram is slower, better on rare words. Guaranteed exam question; students flip them constantly.
Negative sampling
Word2Vec's efficiency trick: rather than a softmax over the whole vocabulary, discriminate the true context word from a handful of random "negative" words. Turns an impossible normalization into cheap binary classification.
GloVe Global Vectors
Embeddings from factorising a global word-word co-occurrence matrix. Contrast with Word2Vec's local sliding window — global counts vs. local prediction is the comparison to teach.

Unit III — Deep learning

RNN Recurrent Neural Network
Processes a sequence one step at a time, carrying a hidden state. Weights shared across timesteps.
Hidden state
The vector an RNN carries forward — its compressed memory of everything seen so far.
Vanishing gradient
Gradients shrink multiplicatively over long backprop paths, so plain RNNs cannot learn long-range dependencies. The disease; LSTM is the treatment.
LSTM Long Short-Term Memory
RNN with a cell state and gates (forget, input, output). The cell state's additive path lets gradients survive.
GRU Gated Recurrent Unit
Simplified LSTM: two gates (reset, update), no separate cell state. Fewer parameters, usually comparable performance.
Seq2seq encoder–decoder
Encoder reads input into a representation; decoder generates output. Enables mismatched input/output lengths — hence machine translation.
Bottleneck problem
Vanilla seq2seq crushes the whole input into one fixed vector. The motivation for attention. Teach the problem before the solution or attention looks arbitrary.
Attention
Lets the decoder look back at all encoder states, weighted by relevance, instead of one summary vector. Originally Bahdanau et al., 2014 — pre-dates transformers.
Attention vs. Self-attention
Attention (cross-): decoder attends to encoder — two sequences. Self-attention: a sequence attends to itself, each token gathering from every other — one sequence. The transformer's core is self-attention. Do not blur them.
Q, K, V query, key, value
Three projections of each token. Query matches against keys to produce weights; weights pool the values. A soft, differentiable dictionary lookup.
Scaled dot-product attention
softmax(QKᵀ / √dk)V. The √dk is the exam question: without it, dot products grow with dimension, pushing softmax into saturation where gradients vanish.
Multi-head attention
Several attention operations in parallel on split projections, concatenated. Lets different heads attend to different relation types at once.
Positional encoding
Self-attention is permutation-invariant — it cannot see order — so position must be injected explicitly. Ask the class why an RNN needs no such thing.
Transformer
Architecture built from self-attention + feed-forward blocks, no recurrence. Parallelises across the sequence, which is why it won. Vaswani et al., 2017.
Encoder-only vs. Decoder-only
Encoder-only (BERT): bidirectional, sees the whole sequence, for understanding — classification, NER. Decoder-only (GPT): causally masked, cannot see the future, for generation. "What's the difference between BERT and GPT?" is the question you will be asked. This row is the answer.
MLM Masked Language Modelling
BERT's pre-training task: mask ~15% of tokens, predict them from both sides. Bidirectional context is only possible because the task is fill-in-the-blank, not next-token.
Pre-training vs. Fine-tuning
Pre-training: expensive, once, self-supervised on unlabelled text. Fine-tuning: cheap, per-task, supervised on a small labelled set. The economics of this split is why modern NLP is accessible to your students at all.
DistilBERT
BERT compressed by knowledge distillation — a small "student" trained to imitate the large "teacher". ~40% smaller, ~60% faster, ~97% of performance. Named in your syllabus (spelled "DitilBERT" — a typo).
Transfer learning
Reusing representations learned on one task for another. The umbrella that pre-train → fine-tune sits under.
